# yorizori_recommend
yorizori 프로젝트의 추천 시스템이다

## yorizori_recommend_modelmaker.py
추천을 위한 모델을 생성하는 파일이다.

이 파일을 실행하면 전체 유저의 인덱스 관리를 위한 txt파일, 추천도 계산을 위한 csv파일, 유저 개개인 로그 등의 정보를 취합한 csv파일 3종이 외부 파일로 저장된다.

## yorizori_recommend.py
상기한 3가지 파일을 이용하여 한 사용자에 대해 추천할 recipe_id를 반환하는 파일이다.

파일 내부의 recommend_userid를 참고하여 추천을 진행한다.


### 추후 변경할 내용

1. modelmaker.py에서 전체 사용자를 대상으로 연산하는데, recommend에서 개인 사용자의 age, gender가 같은 사용자끼리 연산을 할 수 있는 방법이 있을지 찾아보고 변경
 (문제 : 이미 전체 사용자에 대해 svds가 진행된 파일인데 여기서 일부 사용자끼리 merge해서 판단한다 해도 recipe_id별 predictions score은 전체 사용자인데 그럼 maker에서부터 다시 만들었어야 하는가?)

2. read_csv를 통한 database에서 추출한 정적인 데이터를 사용하고 있다. 모델 갱신을 하루에 한번 진행한다고 가정하고 진행했는데, 실시간 갱신을 하고싶다면 database에 연결해서 df를 구성하는 식으로 수정 요함.

3. 검증을 위한 데이터의 신뢰성 이슈 (로그 데이터를 난수로 넣었기 때문에 신뢰성의 문제가 발생. 검증하는데 타당성 부여하기에 근거가 부족하다. 사용자별 한식위주 양식위주로 범주화 해 조회하는 식으로 신뢰성을 올려야 한다.)
